# 系统架构

## 三服务器拓扑

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   新加坡 DO     │     │    阿里云 ECS    │     │   AutoDL GPU    │
│  4GB / 2核      │◄───►│  7GB            │◄───►│  RTX 4090       │
│  9 Docker容器   │     │  LoRA 推理+数据  │     │  按需训练        │
│  ONNX CPU 推理  │     │  SSH 隧道 5075   │     │  训练完即关机    │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

## 三层模型架构

### Layer 1: ONNX 哨兵（新加坡 CPU）
- **延迟**: 10ms/条
- **模型**: DistilBERT 量化 ONNX（256KB~130MB）
- **任务**: 情绪分类、新闻分类、相关度判定
- **特点**: 极低延迟，全量初筛，日处理 5000+ 条

### Layer 2: LoRA 榫卯（阿里云）
- **延迟**: 2-60s/条
- **基座**: Qwen1.5-1.5B
- **任务**: 精细相关评分、紧急度判断
- **特点**: 需要基座模型（~3GB），精度高

### Layer 3: API 深度分析
- **延迟**: 500ms-5s/条
- **模型**: DeepSeek / MiMo
- **任务**: 个性化投资建议、市场情绪指数、风险预警
- **特点**: 最高质量但按量付费

## 数据流

```
采集(trendradar)
  │
  ├─→ SQLite (原始数据)
  │
  ├─→ ONNX 哨兵 (情绪/分类/相关)
  │     │
  │     ├─→ LoRA 精排 (相关/紧急度)
  │     │
  │     └─→ 任务队列 (queue.json)
  │
  ├─→ 决策引擎 (8因子加权)
  │
  ├─→ 执行引擎 (状态机)
  │     │
  │     ├─→ 钉钉推送
  │     │
  │     └─→ 反馈学习 (模式识别)
  │
  └─→ Knowledge Base (向量数据库)
```

## 决策引擎

8 因子加权评分：

- 技术分析: 20%
- 情绪分析: 15%
- 多时间框架: 15%
- 动量指标: 15%
- 历史模式: 10%
- 波动率: 10%
- 市场情绪: 10%
- 关键词: 5%

买入阈值: ≥0.58 | 卖出阈值: ≤0.42
